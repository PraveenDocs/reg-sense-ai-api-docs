This glossary defines the key technical and regulatory terminology used throughout the Reg-Sense AI API documentation.

## API Key

A unique, securely generated alphanumeric string that authenticates a user or application to the Reg-Sense AI service. It is acquired via the Reg-Sense AI Customer Portal and must be passed as a **Bearer Token** in the HTTP request header.

## Clinical Trial Data

Structured data collected during phases of pharmaceutical research (e.g., patient demographics, adverse event records, lab results). This data is the primary input for the Reg-Sense AI ingestion endpoint.

## ICD-10

International Classification of Diseases, 10th Revision. A standardized set of medical diagnosis and procedure codes used globally to report and track health information.

## PII (Personally Identifiable Information)

Any data that can be used to identify, contact, or locate an individual, such as name, address, email, or patient ID. Compliance auditing is mandatory to ensure PII is handled securely according to GDPR and HIPAA standards.

## Regulatory Compliance

The process of ensuring all clinical trial data structures and reporting methods adhere to the mandates set by government agencies (e.g., FDA, EMA).
